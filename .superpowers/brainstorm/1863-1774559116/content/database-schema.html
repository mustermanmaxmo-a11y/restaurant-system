Datenbankschema

Multi-Tenant — alle Restaurants in einer Datenbank, isoliert per restaurant_id + RLS

KERN — TENANTS & AUTH
🏢 restaurants
id uuid PK
owner_id uuid → auth.users
name text
slug text UNIQUE
plan text 'basic' | 'pro'
stripe_customer_id text
stripe_subscription_id text
active boolean
created_at timestamptz
👤 staff
id uuid PK
restaurant_id uuid → restaurants
name text
code text (4-PIN)
role text 'kitchen' | 'waiter'
active boolean
created_at timestamptz
🪑 tables
id uuid PK
restaurant_id uuid → restaurants
table_num integer
label text (z.B. "Terrasse 3")
qr_token text UNIQUE
active boolean
created_at timestamptz
MENÜ
📂 menu_categories
id uuid PK
restaurant_id uuid → restaurants
name text
sort_order integer
active boolean
🍔 menu_items
id uuid PK
restaurant_id uuid → restaurants
category_id uuid → menu_categories
name text
description text
price numeric(10,2)
image_url text
allergens text[] (Array)
tags text[] (vegan, vegetarisch...)
available boolean
sort_order integer
BESTELLUNGEN
📦 orders
id uuid PK
restaurant_id uuid → restaurants
order_type text 'dine_in' | 'delivery' | 'pickup'
table_id uuid → tables (nullable, nur dine_in)
status text 'new' | 'cooking' | 'served' | 'cancelled'
items jsonb [{item_id, name, price, qty}]
note text
total numeric(10,2)
customer_name text (delivery/pickup)
customer_phone text
delivery_address jsonb (nullable)
estimated_time integer (Minuten)
created_at timestamptz
🔔 service_calls
id uuid PK
restaurant_id uuid → restaurants
table_id uuid → tables
type text 'waiter' | 'bill'
resolved boolean
created_at timestamptz
🔒
Row Level Security (RLS)
Jede Tabelle hat RLS aktiviert. Restaurant-Owner sehen nur ihre eigenen Daten. Staff sieht nur Daten ihres Restaurants. Gäste können nur lesen/schreiben was ihr Token erlaubt. Kein Restaurant kann Daten eines anderen sehen — auch wenn sie dieselbe DB teilen.
